Metropho.rs

2011

Picking out geographic metaphors from twitter and using them to build a parallel geography. More...

In June of 2012 I got curious about the fact that my hometown of Detroit used to be known as "the Paris of the Midwest". In particular I was curious about how else this snowclone was used. To answer that question I built metropho.rs as a side project. It was a lot of fun to work on and got a nice reception from The Atlantic Monthly's Atlantic Cities Blog , the SFGeo meetup, and this Chinese Google Maps fanblog. It's a Django json api consumed by a knockout/modestmaps front end. It's hosted on AWS EC2 and backed by a AWS RDS db. It uses twython to troll Twitter and Yahoo's Placemaker to extract place names from tweets. I made the backing map in Tilemill and serve it with Tilestache.

Gap Inc. Purchase Order Management System

2011-2013

Helping the largest specialty retailer in the U.S. manage purchase orders and save money. More...

We replaced a legacy app at Gap for managing Purchase Orders. Our new system removes a whole set of error prone manual copy and pasting steps and lets Gap consolidate similar orders across all their brands and international markets. The app we built was a rich javascript front end working with a Java and Mongo backend that served JSON. The front end was written with knockout.js, which allowed us to completely separate our JS from the DOM and write somewhere around 900 headless jasmine specs that ran in under 70 seconds. We did data validation with a "write-once, run on either end" javascript validation engine that ran server side with Rhino. The majority of our automated testing hit the API level, where we had 900 JRuby browserless cucumber scenarios that ran in 7 minutes. All under a Jenkins setup that included one click deployment. Martin Fowler wrote up a nice presentation on the project here.

Content Validation UI For Clinical Decision Support Provider

2010-2011

Instantaneous feedback mechanism to let users know when they would fail at integration, massively reducing wasted time, effort and aggravation. More...

My first project at ThoughtWorks was for a client in Los Angeles that provided Clinical Decision Support (sort of like Best Practices checklists for medical conditions.) Their users were having a hard time exporting content into the myriad systems hospitals used to track and update this data. We worked with them over the course of a year to add a feature (built in javascript and ASP.net) that would alert users what data would and would not make it into their downstream system. Along the way we got them to adopt pair programming, TDD, CI, fearlessly releasing frequently and continuous refactoring. We turned an arduous, dangerous, weekend-long release process into something that was a non-event after lunch anytime it was deemed necessary. We took an app that had no tests and a six month long manual test cycle into a well covered (mstest and sahi driven through Twist), Jenkins'd source of confidence for everyone. We also managed to convince them to ditch TFS for git, which was a proud day for all involved parties.
